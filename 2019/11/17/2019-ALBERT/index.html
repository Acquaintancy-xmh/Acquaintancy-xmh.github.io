2019-ALBERT
Created 2019-11-17| Updated 2019-11-17
Post View:

Abstract

  1. 更大的模型通常会提升性能，但是也会受到GPU的限制、需要更长的训练时间和可以造成的模型degradation。
  2. 提出了两种parameter-reduction技术，降低计算复杂度提升BERT的训练速度。
  3. 用self-supervised loss去专注于建模句子间的连贯性，并表明它始终有助于多句子输入的下游任务。

Introduction

  1. 构建更大的模型的一个障碍是可用硬件的内存限制。考虑到目前最先进的模型通常有数亿甚至数十亿个参数，当我们试图扩展模型时，很容易遇到这类限制。在分布式训练中，训练速度也会受到很大的影响，因为通信开销与模型参数的数量成正比。

    我们还观察到，简单地增加模型的 hidden size 可能会导致性能下降，比如 BERT-large。我们简单地将这个 BERT-xlarge 模型的 hidden size 增加到 2 倍，结果却很糟糕。

  2. 第一个技术是对嵌入参数化进行因式分解(factorized embedding parameterization)。通过将大的词汇表嵌入矩阵分解为两个小的矩阵，将隐藏层的大小与词汇表嵌入的大小分离开来。这种分离使得在不显著增加词汇表嵌入的参数大小的情况下，更容易增加隐藏层大小。

  3. 第二种技术是跨层参数共享(cross-layer parameter sharing)。这种技术可以防止参数随着网络深度的增加而增加。

  4. 提出了self-supervised loss for sentence-order prediction（SOP）用来解决NSP不ok的问题。

Model Architecture Choices

基本框架和BERT类似（Transformer encoder ➕ GELU）

Factorized embedding parameterization

  1. 在BERT（XLNet/RoBERTa）中，词典embedding == hidden layer size

  2. 在large和xlarge等更大规模的模型中，E会随着H不断增加，作者认为没有这个必要，因为预训练模型主要的捕获目标是H所代表的“上下文相关信息”而不是E所代表的“上下文无关信息”。

  3. 在词表V到隐藏层H的中间，插入一个小维度E，多做一次尺度变换：

    O(V*E + E*H)

  4. 但是词向量压缩并不能减少太多的参数量。

Cross-layer parameter sharing

  1. 多种共享参数的方法：只共享feed-forward network参数或者只共享attention的参数。ALBERT的默认是共享所有参数。（相当于直接把12个完全相同的层摞起来）
  2. 效果好不好……见后面实验吧

Inter-sentence coherence loss

  1. RoBERTa就证明了NSP是废的，不work
  2. NSP过于简单了，把topic prediction和coherence prediction融合了起来。
  3. 正例是两个连续的句子（同NSP），负例是把两个连续句子的顺序调换，这样的话就排除了topic prediction，只关注与coherence prediction。
